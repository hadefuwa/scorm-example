# Course Build Prototype

A pipeline and site prototype that turns one tagged Word document into multiple checked course outputs — learner HTML pages, worksheet outputs, reports, and optionally a SCORM package.

## What this is

The core idea is simple: **one source document, one set of rules, many outputs** — generated by AI agents working through a defined set of stages.

The prototype site demonstrates the complete pattern with real evidence: a tagged DOCX is analysed to find 32 blocks, two blocks are fully extracted and built into real outputs, and the SCORM packaging stage is safely blocked until the required decisions are explicitly defined.

**Site pages:**

| Page | Purpose |
|---|---|
| `index.html` | What the project does, with an animated SVG pipeline diagram and SCORM explainer |
| `workflow.html` | Engineer prep steps and the 5-stage AI pipeline, visualised |
| `backend.html` | Source tagging rules, filename rules, common errors, SCORM requirements, and repo proof |
| `upload.html` | Browser-based upload demo |

## How the pipeline works

### 1 — Engineer prepares the source

Write course content in Word, then mark each block with tags so the AI knows what to build:

```
<HTML>
Homework 1
<filename>"CP4807-H1.html"</filename>

Design a program that controls two sets of traffic lights at a junction.
</HTML>
```

**Tag types:**

| Tag | Builds | SCORM |
|---|---|---|
| `<HTML>` | Learner-facing HTML page | Assessment subtype becomes a SCO item |
| `<worksheet>` | Worksheet output | Not tracked |
| `<document>` | Reference document | Not tracked |

**Filename rules:**
- Must be unique across the whole source document
- Must include a file extension
- Must be wrapped in double quotes inside `<filename>` tags
- Extension should match the tag type (e.g. `<HTML>` → `.html`)

**Common errors that cause blocks to be skipped:**
- Duplicate filename — the second block is flagged and not extracted
- Missing file extension — the pipeline cannot route the output
- Mismatched closing tag casing — `<HTML>` must close with `</HTML>`, not `</html>`

### 2 — AI processes the source

Five sequential stages:

| Stage | Input | Output |
|---|---|---|
| **Analyse** | Tagged DOCX | `reports/analysis-report.md` — blocks, filenames, warnings, SCORM gaps |
| **Extract** | Analysed source | `outputs/extracted/` — clean text files per block |
| **Build** | Extracted files | `outputs/html/` and `outputs/docx/` — learner outputs |
| **Validate** | Built outputs | `reports/validation-report.md` — missing files, broken refs |
| **Package** | Validated outputs + SCORM decisions | SCORM package or safe stop |

### 3 — Outputs produced

- `outputs/html/` — learner-facing HTML pages
- `outputs/docx/` — worksheet and document outputs
- `reports/` — one report per pipeline stage
- SCORM package only when all five required decisions are defined

---

## SCORM packaging requirements

Packaging is blocked until all five decisions are explicitly defined in the source or a companion definitions file. None can be guessed or defaulted.

| # | Decision | Example |
|---|---|---|
| 1 | SCORM version | `scorm-version: "1.2"` |
| 2 | Launch file | `launch: "CP4807-H1.html"` |
| 3 | Tracking model | `tracking: "completion-only"` |
| 4 | Completion model | `completion: "page-view"` |
| 5 | Navigation rules | `navigation: "free"` |

If any are missing, the packaging report lists exactly what is absent and the stage stops safely.

## Project layout

```
source/           — master Word source document(s)
outputs/
  extracted/      — clean block text files
  html/           — learner-facing HTML pages
  docx/           — worksheet and document outputs
  browser-course/ — browser course demo
  scorm/          — SCORM packages (when produced)
reports/          — analysis, extraction, build, validation, packaging reports
specs/            — tagging rules, output rules, SCORM rules
agents.md         — agent behaviour and stop conditions
prompt-01-analyse.md
prompt-02-extract.md
prompt-03-build.md
prompt-04-package-scorm.md
```

## Rules files

- `specs/tag-rules.md` — block recognition and filename binding
- `specs/output-rules.md` — output behaviour and naming conventions
- `specs/scorm-rules.md` — minimum SCORM requirements and stop conditions
- `agents.md` — agent roles and safe-stop conditions
