How It Works

Two halves: the engineer prepares the source, then the AI processes it.

The workflow is easiest to understand when human preparation and AI automation are shown as separate parts of the same pipeline.

At A Glance

Engineer prepares source -> AI processes source -> outputs produced.

Engineer prepares source

Write, tag, name, and define the source properly.

->

AI processes source

Analyse, extract, build, validate, and package if safe.

->

Outputs produced

HTML, worksheet output, reports, and SCORM only when ready.

What The Engineer Does

Prepare the source so the AI has something safe and structured to work from.

1

Write in Word

Create the course, homework, worksheet, and assessment content in one source document.

Result: one master authoring file
2

Add tags

Mark blocks with tags like <HTML>, <worksheet>, and <document>.

Result: the AI knows what type of output each block should become
3

Add filenames

Use <filename> tags so each block has a clear target name.

Result: outputs can be built into the right folders with the right names
4

Define SCORM decisions

Add version, launch, tracking, completion, and navigation details if SCORM packaging is required.

Result: packaging can happen safely later instead of being guessed
5

Save into the repo

Place the prepared file in the source folder for the pipeline to use.

Result: the AI has a defined input, such as source/CP4807 - Introduction to microcontrollers 05 04 26.docx

What The AI Does

Take the prepared source and transform it into checked outputs.

Analyse

Read the source

Input: tagged DOCX.

AI does: finds blocks, filenames, warnings, and missing SCORM decisions
Output: reports/analysis-report.md
Extract

Pull out valid blocks

Input: analysed source blocks.

AI does: creates clean extracted text files
Output: outputs/extracted/
Build

Create learner outputs

Input: extracted block files.

AI does: builds learner HTML and worksheet/document output
Output: outputs/html/ and outputs/docx/
Validate

Check the result

Input: built outputs.

AI does: checks missing files, references, duplicates, and unresolved items
Output: reports/validation-report.md
Package

Attempt SCORM if safe

Input: validated outputs plus SCORM decisions.

AI does: packages only if launch, version, and tracking details are explicit
Output: SCORM package or safe stop

Final Note

The engineer controls the source. The AI handles the repetitive transformation work.

The source, tags, filenames, and SCORM definitions come from the engineer. The AI handles the repeated analysis, extraction, building, validation, and packaging steps. SCORM only happens when enough information exists.
