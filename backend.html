Prompts And Rules

The prompts line up directly with the stage model.

Prompt files

  • `prompt-01-analyse.md` finds blocks, filenames, warnings, and SCORM gaps.
  • `prompt-02-extract.md` creates the staged block files.
  • `prompt-03-build.md` creates learner-facing outputs.
  • `prompt-04-package-scorm.md` stops packaging if the required SCORM decisions are missing.

Rules files

  • `tag-rules.md` defines block recognition and filename binding.
  • `output-rules.md` defines output behavior and naming.
  • `scorm-rules.md` defines minimum requirements and stop conditions.
