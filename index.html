About SCORM

What SCORM is — and why it changes what is possible.

Create once.
Deliver anywhere.

SCORM makes digital learning portable. It is the standard that lets online courses work inside compatible learning platforms — so content can be launched, tracked, and reused across systems without rebuilding from scratch.

What SCORM is

The standard that makes digital learning portable. SCORM lets online courses work inside compatible platforms so content can be launched, tracked, and reused more easily across systems.

One standard. Many compatible platforms.

What makes a SCORM package

Great content is only part of the picture. A SCORM package also needs launch information, package metadata, and the rules that tell a learning platform how to handle the content.

Content + structure + rules = a real package.

Why the source document is powerful

A tagged Word source can become the starting point for multiple polished outputs — learner-facing pages, structured course assets — while also preparing the ground for SCORM packaging later on.

One source. Many possibilities.

Why the source is not SCORM on its own

The Word file holds the content and intent. SCORM adds the delivery structure around it — so the content can be launched, organised, and tracked inside an LMS. One is the beginning, the other is the finished package.

Source = content. SCORM = content + delivery structure.

Open packaging report

What the tags provide

  • <HTML> → resource type is webcontent, not a binary asset.
  • <filename> → exact href and identifier, no guessing required.
  • Classification "assessment" → SCO type, so the LMS can track completion.

What the tags cannot provide

  • SCORM version (1.2 vs 2004) — must be decided by the author.
  • Completion model — not derivable from content tags alone.
  • Navigation rules — free vs linear must be explicitly specified.
